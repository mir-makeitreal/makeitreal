## 2. Changes (in implementation order)

### Change 1: Dashboard Renderer Refactor [HIGHEST VISUAL IMPACT]

**Problem:** render-dashboard-html.mjs is 4018 lines in one file. Information
hierarchy is flat — everything at same density. 12000px single scroll.
Diagrams are unclear inline SVGs.

**Solution:** Split into modular template-literal renderers + Mermaid pre-render.

```
src/preview/
├── render-dashboard-html.mjs    ← Shell: layout, head, nav (200 lines)
├── sections/
│   ├── hero.mjs                 ← Status + title + 4 metric cards + next action
│   ├── system-map.mjs           ← Module topology (pre-rendered Mermaid SVG)
│   ├── task-breakdown.mjs       ← DAG diagram + work item cards (merged from Task DAG + Worker Topology)
│   ├── contracts.mjs            ← Interface signatures with syntax coloring
│   ├── responsibility.mjs       ← Ownership boundaries with file trees
│   ├── scenarios.mjs            ← Sequence diagrams (pre-rendered Mermaid SVG)
│   ├── review.mjs               ← Decision log table
│   ├── verification.mjs         ← Checklist-style evidence
│   └── diagnostics.mjs          ← Generation metadata, sources
├── styles/
│   └── dashboard.mjs            ← CSS as template literal (dark/light mode, CSS custom props)
├── charts/
│   ├── mermaid-render.mjs       ← Generate Mermaid text from preview-model data
│   └── svg-inline.mjs           ← Fallback: direct SVG generation (current approach, kept)
└── layout/
    ├── sidebar.mjs              ← Sticky TOC with section links
    └── progressive-disclosure.mjs ← details/summary wrappers
```

**Key design decisions:**
- NO React, NO Vite, NO build step. Template literals → single HTML file.
- Mermaid diagrams pre-rendered to SVG at generation time via @mermaid-js/mermaid-cli
  - If mmdc unavailable: fall back to current inline SVG generation (no hard dependency)
- Dark mode by default, light mode toggle (CSS custom properties)
- Progressive disclosure: details/summary for all sections below hero
- Sticky sidebar TOC with IntersectionObserver scroll-spy (~15 lines JS)
- Target: < 200KB output HTML
- First viewport: status badge, title, 4 metric cards, compact architecture diagram

**Section order (based on developer reading priority):**
1. Hero (always visible): status, title, metrics, next action
2. System Map (always visible): compact topology diagram
3. Task Breakdown (auto-expanded): DAG + work item cards
4. Contract Surfaces (collapsed): interface signatures
5. Responsibility Map (collapsed): ownership + file trees
6. Scenarios (collapsed): sequence diagrams
7. Review Decisions (collapsed): decision log
8. Verification (collapsed): evidence checklist
9. Diagnostics (collapsed): metadata, sources

**Rendering contract (UNCHANGED):**
```
INPUT:  preview-model.json (generated by preview-model.mjs — no changes)
OUTPUT: preview/index.html (self-contained, works as file:/// URL)
API:    renderDesignPreview() returns {ok, previewDir, errors} — no changes
```

**Test strategy:** Existing render-preview tests check {ok, previewDir, errors}.
Dashboard content tests: add snapshot tests for each section renderer.

### Change 4: Source Deduplication [CLEANUP]

**Current:** `src/` and `plugins/makeitreal/dev-harness/src/` are identical copies.
`npm run plugin:sync` maintains the copy.

**Proposed:** Keep the copy strategy (symlinks may break Claude Code plugin resolution)
but add a CI check:

```json
// package.json scripts
"plugin:check": "diff -rq src/ plugins/makeitreal/dev-harness/src/ && echo 'Sources in sync'"
```

Add to `release:check` pipeline. Low risk, low effort.

## 3. Dropped Changes

- **preview-model.json deduplication (was Change D):** Dropped per reviewer consensus.
  It's a generated intermediate file consumed by one module. Duplication costs nothing.
  Adding a ref system adds bugs. Let the dashboard handle the flat model.

- **React + Vite dashboard (was Change A):** Dropped per all 3 reviewers.
  Build step dependency, bundle size, file:/// issues. Template literal refactoring instead.

- **domain-profile.json as user config (was part of Change B):** Dropped.
  Users won't configure field-type heuristics. Keep internal, expose only if asked.
