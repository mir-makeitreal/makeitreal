Contracts

Contract Surfaces

Each public surface is shown as an SDK-style schema. Consumers should rely on this contract, not provider implementation details.

Auth UI · ru.frontend

component-event

LoginForm.submit

contract.auth.login

LoginForm.submit(credentials.email, credentials.password): AuthSession

Submits credential input and delegates authentication through the declared auth login contract.

Inputs

  • credentials.email string · required

    User email address collected by the login form.

  • credentials.password string · required

    User password collected by the login form.

Outputs

  • sessionResult AuthSession

    Session response returned by contract.auth.login.

Errors

  • AUTH_LOGIN_REJECTED The auth service rejects the credential payload. · Surface the declared auth error state; do not infer fallback session behavior.
Usage Example Declared surface only 
// LoginForm.submit is the declared public surface owned by Auth UI.
// Call it only through the Blueprint contract; do not read implementation internals.
const sessionResult = LoginForm.submit("user@example.com", "correct horse battery staple");

Auth Service · ru.auth-service

http

POST /auth/login

contract.auth.login

POST /auth/login -> AuthSession

Authenticates credentials and returns the declared session response.

Inputs

  • request.body.email string · required

    Email value from the auth UI.

  • request.body.password string · required

    Password value from the auth UI.

Outputs

  • 200.session AuthSession

    Authenticated session payload defined by the OpenAPI contract.

Errors

  • 401.INVALID_CREDENTIALS Credentials are syntactically valid but authentication fails. · Return the declared error response; do not issue partial sessions or fallback identities.
Usage Example Declared surface only 
const requestBody = {
  email: "user@example.com",
  password: "correct horse battery staple"
};

const httpResponse = await fetch("/auth/login", {
  method: "POST",
  headers: { "content-type": "application/json" },
  body: JSON.stringify(requestBody)
});

if (!httpResponse.ok) {
  throw new Error(`HTTP ${httpResponse.status}`);
}

const session = await httpResponse.json();

Human Review

Review Decisions

  1. Auth UI owns apps/web/auth/** and exposes LoginForm.submit.
  2. Auth Service owns services/auth/**, contracts/auth-login.openapi.json and exposes POST /auth/login.
  3. Auth UI may call Auth Service only through contract.auth.login.
  4. contract.auth.login is reviewed from contracts/auth-login.openapi.json.
