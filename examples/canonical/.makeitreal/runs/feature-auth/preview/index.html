LoginForm.submit

LoginForm.submit(credentials.email, credentials.password): AuthSession

Submits credential input and delegates authentication through the declared auth login contract.

Parameters 2
Returns 1
Errors 1
Contracts contract.auth.login

Consumers: Auth page composition, browser interaction tests

Parameters

credentials.emailstring · required · User email address collected by the login form.
credentials.passwordstring · required · User password collected by the login form.

Returns

sessionResultAuthSession · Session response returned by contract.auth.login.

Errors

AUTH_LOGIN_REJECTEDThe auth service rejects the credential payload. · Surface the declared auth error state; do not infer fallback session behavior.
// LoginForm.submit is the declared public surface owned by Auth UI.
// Call it only through the Blueprint contract; do not read implementation internals.
const sessionResult = LoginForm.submit("user@example.com", "correct horse battery staple");

Imports 0

http

POST /auth/login

POST /auth/login -> AuthSession

Authenticates credentials and returns the declared session response.

Parameters 2
Returns 1
Errors 1
Contracts contract.auth.login

Consumers: ru.frontend

Parameters

request.body.emailstring · required · Email value from the auth UI.
request.body.passwordstring · required · Password value from the auth UI.

Returns

200.sessionAuthSession · Authenticated session payload defined by the OpenAPI contract.

Errors

401.INVALID_CREDENTIALSCredentials are syntactically valid but authentication fails. · Return the declared error response; do not issue partial sessions or fallback identities.
const requestBody = {
  email: "user@example.com",
  password: "correct horse battery staple"
};

const httpResponse = await fetch("/auth/login", {
  method: "POST",
  headers: { "content-type": "application/json" },
  body: JSON.stringify(requestBody)
});

if (!httpResponse.ok) {
  throw new Error(`HTTP ${httpResponse.status}`);
}

const session = await httpResponse.json();
